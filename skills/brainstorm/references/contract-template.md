# contract.json Template

Pair this with `contract.schema.json`.

Keep JSON keys machine-readable and unchanged. Use `.vibeRig/project.yaml`
`output.language` for human-readable string values such as `title`, goal text,
rule descriptions, risk descriptions, and decisions.

```json
{
  "requirement_id": "REQ-001",
  "title": "",
  "goals": [],
  "non_goals": [],
  "rules": [],
  "entities": [],
  "workflows": [],
  "constraints": [],
  "risks": [],
  "decisions": []
}
```
